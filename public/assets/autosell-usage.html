一、回复内容如何设置

路径：侧边栏 → 回复内容

回复内容用于存放自动回复要发送的文字或图片。在「自动回复」规则里通过多选这些条目，按顺序逐条发送给买家。此处只做「内容库」的维护，不涉及发货。

1.1 文字回复内容

  • 点击「添加文字」，在弹窗中填写：名称（便于在自动回复里识别）、内容（要发送的正文）、说明（可选）、适用账号（留空表示全部账号可用）、启用（勾选后才会在自动回复的下拉中出现）。
  • 保存后，该条会出现在「文字回复内容」列表中；在自动回复页添加/编辑规则时，「回复内容」下拉里会按名称列出，可多选多条，将按选中顺序逐条发送。
  • 支持搜索、批量选择、批量删除/启用/禁用；可导出模板 CSV、导入 CSV 批量新建。
【截图示例：回复内容页 - 文字回复列表与「添加文字」弹窗】

1.2 图片回复内容

  • 点击「添加图片」，在弹窗中填写：名称、上传图片、说明、适用账号、启用。
  • 保存后，在自动回复规则中可像文字回复一样多选「图片回复内容」，与文字回复可混合多选，按顺序发送（先发的文字/图片在前）。

小结：回复内容页只负责维护「文字条」和「图片条」；具体何时发、匹配什么关键词，在自动回复页的规则里配置。

二、发货内容如何设置

路径：侧边栏 → 发货内容

发货内容页管理三类素材：固定文本、库存发货、API 取货，供「自动发货」规则或「发货流程」中的发货节点使用。

2.1 固定文本

  • 点击对应卡片的「添加」，填写名称、文本内容（卡密、链接等）、可选图片与说明、适用账号、启用；可选商品规格（如 颜色/红色），在自动发货中选择该条时，规则中的「商品规格」会自动带入。

2.2 库存发货

  • 添加后填写名称、库存内容（多行，每行一条，发一单消耗一行）、可选「使用说明」（会随每条库存一起发给买家）、适用账号、启用；可选商品规格。

2.3 API 取货

  • 添加后配置请求 URL、方法、请求头/体、响应取值字段；适用账号、启用；可选商品规格。

三、自动发货页面如何设置

路径：侧边栏 → 自动发货

自动发货页用于配置规则：当订单满足条件时，用哪类发货内容、是否走某条「发货流程」。

3.1 逐步设置说明

  1. 规则名称（必填）：如「卡密自动发货」，便于在右侧列表识别。
  2. 筛选账号（可选）：多选后，下方「选择商品」只显示这些账号下的商品；不选则显示全部账号商品。
  3. 选择商品（必选）：可多选，表示这些商品走本规则。可搜索商品名称或 ID。
  4. 商品规格（可选）：留空表示匹配该商品所有规格；填写如 颜色/红色 则只匹配该规格订单。若在「选择发货内容」里选了已配置规格的发货内容，此处会自动带入该规格。
  5. 触发时机：如「待发货」时触发、「待刀成」时免拼发货等，按业务选择。
  6. 发货流程（可选）：
    • 选「默认流程」：支付后直接 触发→发货→标记发货。
    • 选某条已创建的流程：将按该流程执行（可含延迟、条件、关键词匹配、通知等）。
    • 只要选了非默认的流程，会出现两项：
      「关键词匹配节点 · 发送消息」：仅用于流程里的「关键词匹配」节点。系统在等待买家回复前，会先给买家发这条消息（如「请回复「确认」继续发货」）。此处填写的内容即「关键词匹配」节点要发的那句话。
      「通知节点 · 消息」：仅用于流程里的「通知」节点。若该通知节点在流程画布里未填消息，则用此处内容作为通知文案。
  7. 选择发货内容：下拉中按「固定文本」「库存发货」「API 取货」分组，可多选；选中的在下方按顺序显示。也可不选：固定文本在下方手填、库存在下方填库存、API 在下方填地址。
  8. 根据所选发货类型，在表单下方填写对应内容（手填发货内容、库存行、或 API 配置等）。
  9. 点击「保存」，规则出现在右侧列表，可编辑、启用/禁用、删除。

四、流程中每种节点的使用

路径：侧边栏 → 发货流程 → 选择某流程进入画布

画布根节点为「订单触发」，选中任意节点后右侧出现配置面板，可改节点名称、节点类型及该类型下的参数。以下说明每一种节点在界面中如何配置、以及含义。

4.1 触发节点

作用：订单支付成功时进入该流程，每个流程有且仅有一个触发节点（根节点）。

如何配置：无需配置参数，只需保留为流程入口即可。可在配置面板中修改节点名称（如「订单支付成功」）。

4.2 条件判断节点

作用：根据条件表达式的结果走不同分支：条件成立走「IF: 条件成立」下的子节点，不成立走「ELSE: 条件不成立」下的子节点。

如何配置：在右侧「条件表达式」输入框中填写表达式，例如：

  • order.price > 100 — 订单金额大于 100 时走 IF
  • order.buyerMessage.includes("加急") — 买家留言包含「加急」时走 IF
  • order.quantity > 1 — 商品数量大于 1 时走 IF

配置面板提供「快速插入」按钮（商品价格、商品数量、商品名称包含、买家留言包含、商品ID、买家昵称），点击后会在表达式中插入对应模板，再补全运算符和数值即可。

使用示例：见下文「示例：按金额分支」。

4.3 发货节点

作用：向买家发送发货内容（卡密、链接等）。

如何配置：流程里无需再选内容。实际发什么由「自动发货」页里与该订单商品匹配的那条规则决定（固定文本/库存/API）。只需在画布中添加「发货」类型节点即可，右侧仅有一段说明文字，无输入项。

使用示例：几乎所有流程都会在「触发」或「延迟/条件/关键词匹配」之后接一个「发货」节点。

4.4 标记发货节点

作用：调用平台接口，把订单标记为已发货。

如何配置：在右侧选择「发货方式」：

  • 虚拟发货：确认发货，适用于虚拟商品/无需物流。
  • 免拼发货：免拼发货，适用于拼单商品直接发货。

使用示例：一般在「发货」节点之后添加「标记发货」，完成「发内容 + 平台标记」两步。

4.5 延迟节点

作用：等待一段时间后再执行下一个节点，可模拟人工操作间隔。

如何配置：

  • 延迟模式：选「固定时间」或「浮动时间」。
  • 时间单位：选「毫秒」或「秒」。
  • 若选固定时间：只需填「延迟时间（秒）」或「延迟时间（毫秒）」一个数字。
  • 若选浮动时间：填「最短时间」和「最长时间」，系统在该区间内随机取一个延迟值。

使用示例：见下文「示例：延迟后发货」。

4.6 关键词匹配节点（等待买家回复）

作用：先向买家发送一句提示消息，然后等待买家在对话里回复；当回复内容满足「关键词匹配」条件时，流程才继续执行后面的节点（如发货、标记发货）。

如何配置：

  • 发送给买家的那句话：不在流程里填，而是在「自动发货」页该规则的表单里填——当规则关联了某条流程时，会出现「关键词匹配节点 · 发送消息」。那里填的内容就是本节点要发给买家的提示（如「请回复「确认」继续发货」）。
  • 在流程画布右侧配置面板中只需配置：
    「匹配关键词」：多个关键词用英文斜杠 / 分隔，例如 确认/同意/好的/收到。买家回复中只要命中其中一个即视为通过。
    「匹配模式」：
    · 精确匹配：回复内容必须完全等于某一关键词才通过（如关键词填「确认」，只有回复恰好是「确认」才通过）。
    · 模糊匹配：回复内容包含某一关键词即通过（如关键词填「确认」，回复「好的确认」也会通过）。一般建议用模糊匹配，体验更好。

使用示例：见下文「示例：关键词匹配后再发货」。

4.7 通知节点

作用：发送一条通知消息（如给管理员或外部渠道），不发给买家。

如何配置：在右侧「通知消息」输入框中填写要发送的文案。若此处不填，且该规则在「自动发货」页配置了「通知节点 · 消息」，则使用规则里填的那条消息。

使用示例：见下文「示例：先通知再发货」。

4.8 等待订单完成节点

作用：轮询订单状态，直到订单变为「交易成功」再执行下一节点；若超时则按配置提示结束。

如何配置：轮询间隔（分钟）、最大等待（分钟）、超时提示语（可选）。

五、完整流程示例（含每种节点与关键词匹配）

以下示例按「流程怎么建 + 自动发货规则怎么配」写全，尤其关键词匹配会同时用到「自动发货」里的「发送消息」和流程里的「匹配关键词/匹配模式」。

示例一：只用「触发 → 发货 → 标记发货」

流程：画布中 订单触发 → 添加子节点「发货」→ 再在「发货」下添加子节点「标记发货」；标记发货的「发货方式」选「虚拟发货」或「免拼发货」。

自动发货：规则名称、选择商品、选择发货内容（或手填），发货流程选「默认流程」即可。

示例二：延迟后发货（使用「延迟」节点）

流程：订单触发 → 延迟（名称：延迟5秒；延迟模式：固定时间；时间单位：秒；延迟时间：5）→ 发货 → 标记发货。

自动发货：同上，发货流程选该「延迟5秒发货」流程。

示例三：按金额分支（使用「条件判断」节点）

流程：订单触发 → 条件判断（名称：金额是否大于100；条件表达式：order.price > 100）→ IF 下接：通知（消息填「高金额订单」）→ 发货 → 标记发货；ELSE 下接：发货 → 标记发货。

自动发货：发货流程选该流程；可选填「通知节点 · 消息」作为通知节点默认文案。

示例四：关键词匹配后再发货（使用「关键词匹配」节点）

流程：订单触发 → 关键词匹配（名称：等买家确认；匹配关键词：确认/同意/好的/收到；匹配模式：模糊匹配）→ 发货 → 标记发货。

自动发货：同一规则中，发货流程选该「确认后发货」流程；在「关键词匹配节点 · 发送消息」中填写：请回复「确认」或「好的」继续发货。这样系统会先给买家发这句话，等买家回复里包含「确认」「同意」「好的」「收到」任一词后，再执行发货和标记发货。

关键词匹配要点：「发什么话」在自动发货规则里填，「匹配哪些词、精确还是模糊」在流程里该节点的配置面板填。

示例五：先通知再延迟再发货（使用「通知」+「延迟」）

流程：订单触发 → 通知（名称：新订单通知；通知消息：新订单，即将发货）→ 延迟（固定 3 秒）→ 发货 → 标记发货。

自动发货：发货流程选该流程；若流程里通知节点未填消息，可填「通知节点 · 消息」作为兜底。

示例六：条件 + 关键词匹配 + 通知 + 延迟（综合）

流程：订单触发 → 条件判断（order.quantity > 1）→ IF：通知（多件订单提醒）→ 延迟（3 秒）→ 关键词匹配（关键词：确认/好的，模糊匹配）→ 发货 → 标记发货；ELSE：发货 → 标记发货。

自动发货：规则选该流程；「关键词匹配节点 · 发送消息」填：请回复「确认」继续发货；「通知节点 · 消息」可填：您购买了多件，请确认后发货。
